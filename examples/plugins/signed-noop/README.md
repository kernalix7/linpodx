# signed-noop

Example linpodx plugin demonstrating the **Phase 15 ed25519 signature path**.
The wasm payload itself is a no-op (just the 8-byte module header) — this example
is about the install-time verification flow, not the plugin runtime.

## Files

| File | Purpose |
|--|--|
| `linpodx-plugin.toml` | Manifest. Sets `publisher = "linpodx-test"`. |
| `noop.wasm`           | 8-byte wasm module header. |
| `signature.b64`       | Detached base64 signature over `noop.wasm`. |
| `test.pub`            | PEM (SubjectPublicKeyInfo) for the public verifying key. |
| `test.key.b64`        | The fixed 32-byte ed25519 seed used to generate everything else, base64-encoded. Checked in *only* for reproducibility — **never use this key for real plugins**. |

## Install

```sh
# Trust the test key by copying it into the plugin-keys registry:
mkdir -p ~/.config/linpodx/plugin-keys
cp test.pub ~/.config/linpodx/plugin-keys/linpodx-test.pem

# Detached signature path (daemon picks up signature.b64 automatically):
linpodx plugin install ./

# Or pass overrides explicitly:
linpodx plugin install ./ --signature ./signature.b64 --public-key ./test.pub
```

## Re-signing

The artifacts in this directory were generated by the snippet below. Running it
with the same `SEED` produces byte-identical output.

```python
import base64, os
from cryptography.hazmat.primitives import serialization
from cryptography.hazmat.primitives.asymmetric.ed25519 import Ed25519PrivateKey

SEED = bytes([0x5c] * 32)
priv = Ed25519PrivateKey.from_private_bytes(SEED)

WASM = bytes([0x00, 0x61, 0x73, 0x6d, 0x01, 0x00, 0x00, 0x00])
open("noop.wasm", "wb").write(WASM)

open("test.pub", "wb").write(priv.public_key().public_bytes(
    encoding=serialization.Encoding.PEM,
    format=serialization.PublicFormat.SubjectPublicKeyInfo,
))
open("signature.b64", "w").write(base64.standard_b64encode(priv.sign(WASM)).decode() + "\n")
```

## Bypass (development only)

For unsigned plugins, set `LINPODX_ALLOW_UNSIGNED_PLUGINS=1` on the daemon. The
install will still record a `plugin_signature_rejected` audit row with
`accepted: true` so security review can spot the bypass.
